  • chapter1 | Hugo简介以及静态博客搭建

    导入 ​ 我们都上过互联网, 访问过不同的网页, 像是 淘宝, 京东 之类. 这些网站, 为了提供服务, 需要服务器在云端为它们提供支持, 执行业务, 最后将数据和HTML文档组合在一起返回给浏览器, 这就是最后显示在我们面前的网页. 与此不同的方式是将数据和HTML文档直接放到一起, 托管到第三方平台, 在请求时将数据渲染进页面, 形成了完整的网页, 这种方式被称为Jamstack. ​ Jamstack由于不需要服务器和数据库, 维护网页就变成相当简单了. Hugo是在这种理念指导下出现的一种静态网页生成器, 它可以以简单的方式生成网页, 给不懂编程的人有了拥有自己网页的可能性. 静态网页生成器有很多, 但是Hugo社区最活跃, 文档最详细, 国内使用者也不少, 我们选用Hugo作为搭建自己网站的方式再合适不过了. ​ 如果你迫不及待想看一看最后的效果, 你可以先到我的Hugo 网站一睹为快: 摸鱼天尊| lyt0628 的博客. 目录 使用Hugo+Github搭建静态网页 个性化网页配置 发布和组织内容 一, 使用Hugo+Github搭建静态网页 1.1 准备搭建工具 1.2 下载Hugo并在本地运行 1.3 将网站托管到云端 1.4 访问网站 二, 个性化网页配置 三, 发布和组织内容 3.1 标记语言介绍 3.2 Markdown 3.3 YAML 小结 指南适用条件 本指南在Windows下仅在可以参考, 其中使用到的Scoop是Windows下的包管理工具 本指南需要你可以稳定访问到 https://github.com , 如果访问不到, 可以下载fastgithub(获取使用方式见文末), 或者自行使用魔法
