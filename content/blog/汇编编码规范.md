## Nasm

### 通用

- 语法关键字小写， 如 `mov`
- 寄存器名称大写， 如 `AX, DI, BX`
- 数字， 数值内的字母大写，标识进制的字母小写
- 使用 `0x` 开头表示十六进制， 不使用后缀 h
- 十六进制数和二进制数每隔4位就以`_` 分隔， 如：`0x0000_FFFF`, `0100_1010b`



### 命名规范

---

1. SECTION 命名
   以 `.` 开头， 小驼峰命令。

   正例：

   ```
   [SECTION .boot16] 
   ```

   反例：

   ```
   [SECTION boot16]
   ```

1. 标签命名

  1. 文件可见性的表示地址的标签均以  `LB_` 开头，全部字母大写，不同单词用 `_` 分隔
	
	 如：

	  ```
	  LB_GDT
	  LB_LDT
	  LB_IDT
	  LB_BOOT16
	  ```

  2. 文件可见性的表示结构的标签以 `ENTRY_` 开头， 全部字母大写，不同单词用 `_` 分隔

	 如：

	  ```
	  ENTRY_DESC_BOOT32 ; 段描述符
	  ENTRY_SELECTOR_BOOT32 ; 段选择子

	  ```
3. 变量命名
	
	变量是用 equ 伪指令定义的，不会被分配内存。

	变量均以 `VAR_` 开头，后面使用小驼峰命名。
	
	如：
	
	```
	VAR_gdtLen ; 全局段表长度
	VAR_topOfStack ; 栈顶地址
	```

4. 宏命名
	1. 表示结构体的宏， 大驼峰命令
	
	如：
	
	```
		Descriptor ; 段描述符结构体
		Gate       ； 门描述符结构体
	```
5. 特定场景命名细则



5. 常量

  全部字母大写，语言块用下划线分隔
